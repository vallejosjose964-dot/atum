Instrucciones de uso

ATUM es la interfaz que presenta el cálculo físico del motor THOT. El frontend no altera el resultado: solo envía datos, recibe JSON y lo muestra con gráficos, tabla y consola técnica.

Macro: galaxias y cúmulos. Micro: W boson, Z pole y muon. También se admite un CSV propio del usuario para enviarlo directo al backend.

Ley presentada por el backend: g(r) = G·M(r) / (r² + G·M(r)/a*)

Muestra mínima de la física

1. El vacío responde con una capacidad física fijada por una sola constante τ.

2. En macro, el backend usa M(r) para construir la respuesta efectiva y devolver la curva modelada frente a la observada.

3. En micro, el backend genera niveles discretos del vacío a partir de τ y compara cuál es el más cercano a la partícula objetivo.

4. El frontend solo visualiza esa salida y no reescribe ninguna magnitud.

Muestra mínima del principio físico

Paso 1 · Constante única
El backend fija una escala con τ y desde ahí deriva sus magnitudes efectivas. El frontend no toca esa base.
Paso 2 · Régimen macro
Se toma una masa radial M(r), se calcula la respuesta efectiva y se devuelve una curva modelo para compararla con la observada.
Paso 3 · Régimen micro
Se construyen niveles discretos del vacío y se marca cuál es el más cercano a la masa objetivo de W, Z o muon.
